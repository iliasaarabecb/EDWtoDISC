Preprocesser

@author: aarabil / Aarab Ilias

ESRB | Beyond Banking Team | Securitisation Monitor | EDW Data Production Pipeline 2021

Helper functions and classes to push deal submissions from a local environment to DISC:

This module implements various auxiliary functions/classes which are used to push the latest deal submissions from a local environment to DISC. Adds (minimal) pre-processing to the raw data when pushing to Pipeline 2A to ensure queryable tables in a DISC environment.

Including: * pre-processing of raw data to set the stage for Pipeline 2A

Prerequisite: * JSON file with a dictionary mapping Field Numbers to Variable Names * JSON file with a dictionary mapping Variables to Pandas compatible D-types * JSON file with a dictionary mapping Variables to Impala+Parquet compatible D-types

class Preprocesser:

    def preprocessing(self, df):
        """Applies minimal processing to raw data coming from Pipeline 1 in order to ensure Impala based
        tables can easily work with the data when running Pipeline 2A to Pipeline 3A
        Prerequisites:
        * JSON file with a dictionary mapping Field Numbers to Variable Names
        * JSON file with a dictionary mapping Variables to Pandas compatible D-types
        * JSON file with a dictionary mapping Variables to Impala+Parquet compatible D-types
        Files can be found back in /dictionaries subfolder of the DISC submodule of edwapi. In
        case subfolder is empty, please run create_dictionaries() first from the create_dictionaries
        submodule to get fresh dictionaries.

        Parameters
        ----------
        df : Pandas dataFrame
            Containing raw data based on gzip csv files coming through Pipeline 1

        Returns
        -------
        Pandas dataFrame
            Proccessed dataFrame ready to get injected into Pipeline 2A

        """
        # Define path
        path_root = self._path_root
        path_json = os.path.join(path_root, "dictionaries")

        ## Preprocessing step 0
        # --------------------------
        # EDW/Edwapi issue: when using the download_csv_file() method of a PerformanceClient object
        # it might be that the downloaded csv file has it's second header dislocated to a random row
        # Replication of problem (change destiantion arg to suitable path if needed):
        # pc.download_csv_file(ed_code='RMBSPT000158100620076', data_section='POOL', pool_cutoff_date=dt.datetime(year=int(2014), month=int(8), day=int(29)).date(), destination=r"C:\Users\aarabil\Desktop")
        # Hotfix: Check whether csv_file has a dislocated header, if so -> delete dislocated header
        # and push current (wrong) 2nd header to be the first row

        # Check for dislocated header
        var_poolcutoff = pd.Series(df.AR1.values.flatten())
        idx_dislocated_header = np.where(
            var_poolcutoff.astype(str).str.contains("Pool Cut-off Date") == True
        )
        if idx_dislocated_header[0].size > 0:  # we have a dislocated header

            # dislocate current 2nd header as row
            newRow = pd.DataFrame([j for i, j in df.columns]).astype("object").T
            df = df.droplevel(-1, axis=1)
            newRow.columns = df.columns
            df = pd.concat(
                [newRow, df],
                ignore_index=True,
            )

            # delete dislocated header
            idx = idx_dislocated_header[0] + 1  # idx moved up due to new row
            df.drop(idx, inplace=True)

        ## Preprocessing step 1
        # ----------------------
        # Get rid of *Variable Name* header unless step0 was initiated
        if not idx_dislocated_header[0].size > 0:
            df = df.droplevel(-1, axis=1)
        # No-Data values as indicated by ECB's Reporting templates are converted to Numpy's NaNs
        na_values = ["ND", "ND,1", "ND,2", "ND,3", "ND,4", "ND,5", "ND,6"]
        df.replace(
            to_replace=na_values + [na_value.lower() for na_value in na_values],
            value=np.NaN,
            inplace=True,
        )
        # ND4 sometimes comes with an additional date
        df.replace(to_replace="ND,4*", value=np.NaN, regex=True, inplace=True)
        # Some strings contain comma's (which is our delimiter) and Impala doesn't allow
        # us to change serdeproperties (in contrast to HIVE) so pushing the data into an Impala table would
        # create a mess -> change comma's to semicolons within the data and post-process again when exporting data from DISC
        df = df.apply(lambda x: x.astype(str).str.replace(",", ";").astype("object"))

        ## Preprocessing step 2: data types
        # -----------------------------------
        # Get dictionary of Pandas data types
        with open(os.path.join(path_json, "rmbs_lln_dtypes_pandas.json"), "r") as f:
            data_types_pandas = json.load(f)

        # Convert variables to the right dtype:
        # Pandas has trouble working with Int64 types
        # https://github.com/pandas-dev/pandas/issues/25472
        # Hotfix: jump from object to float64 before mapping to Int64
        for everyVariable in data_types_pandas.keys():
            if data_types_pandas[everyVariable] != "Int64":
                df[everyVariable] = df[everyVariable].astype(
                    data_types_pandas[everyVariable]
                )
            else:
                df[everyVariable] = (
                    df[everyVariable]
                    .astype("float64")
                    .astype(data_types_pandas[everyVariable])
                )

        ## Return processed dataFrame
        # ---------------------------
        return df
